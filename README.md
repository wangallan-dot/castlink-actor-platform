# 幕见 CastLink

面向演员、选角导演、制片公司与经纪机构的 **移动端优先** 数字演员卡和在线选角协作平台。

## 在线预览

GitHub Pages：`https://wangallan-dot.github.io/castlink-actor-platform/`

当前仓库为 **Mobile MVP 0.3**，主设计尺寸为 390 × 844。该版本将演员与项目方之间的项目沟通提升为手机端一级入口。

## 移动端产品结构

- 固定底部五栏导航：首页、演员、角色、消息、我的
- 顶部提供带未读数量的“消息”文字入口，不再使用单独通知铃铛
- 手机安全区适配，兼容全面屏和底部 Home Indicator
- 演员库双列卡片布局
- 角色列表单列信息流
- 手机端底部抽屉筛选
- 演员详情、角色详情和工作台触控优化
- PWA 安装模式，可添加到手机主屏幕
- 轻量离线缓存与独立窗口启动
- 桌面端仍保留完整选角工作台和双栏消息视图

## 沟通中心

- 演员身份与项目方身份切换
- 会话与具体项目、角色绑定
- 未读、待回复、试镜、定角等状态筛选
- 系统截止提醒与双方聊天分开显示
- 会话列表展示项目、角色、当前选角阶段和未读数量
- 手机端会话列表与聊天详情分屏切换
- 桌面端会话列表和聊天窗口同时显示
- 支持文字发送原型
- 支持剧本文件、试镜视频和图片附件展示
- 聊天顶部固定显示当前项目、角色和阶段

## MVP 验证闭环

1. 演员建立标准化数字演员卡；
2. 项目方按角色条件搜索演员；
3. 项目方收藏演员并管理候选人；
4. 双方围绕具体项目和角色进行沟通；
5. 演员接收试镜任务并跟踪进度；
6. 演员资料在浏览器本地保存，验证编辑体验。

## 当前功能

- 移动平台首页与演员、项目方入口
- 演员数据库
  - 姓名、城市、角色气质、技能关键词搜索
  - 城市、性别、年龄、角色标签、认证状态筛选
  - 演员收藏，本地持久化
- 演员详情页
  - 基础资料、形象标签、技能、语言、履历、视频展示
  - 试镜邀请与隐私提示界面
- 角色招募广场
  - 项目类型、城市和关键词筛选
  - 角色详情、报酬、拍摄周期与试镜要求
- 沟通中心
  - 双身份切换
  - 项目角色上下文
  - 会话筛选、未读数和阶段标签
  - 聊天详情和附件展示
- 演员工作台
  - 曝光、收藏、试镜邀请和角色申请概览
  - 试镜进度与资料完整度
- 项目方选角工作台
  - 项目概览
  - 候选演员看板
  - 团队动态和角色完成度
- 演员卡编辑器
  - 基础资料编辑
  - 浏览器 `localStorage` 保存
  - 照片、视频、履历等后续模块入口

## 安装到手机桌面

### iPhone / iPad

1. 使用 Safari 打开在线预览地址；
2. 点击底部“分享”；
3. 选择“添加到主屏幕”；
4. 从桌面点击“幕见”图标启动。

### Android

1. 使用 Chrome 打开在线预览地址；
2. 点击右上角菜单；
3. 选择“安装应用”或“添加到主屏幕”；
4. 从桌面独立启动。

## 当前数据边界

当前消息中心是可交互前端原型：可以切换会话、查看附件和在当前浏览器内追加消息，但尚未接入真实账号、服务器数据库、WebSocket、消息推送或跨设备同步。

当前版本整体仍采用演示数据和浏览器本地存储，没有接入真实身份认证、图片视频上传、在线合同和支付。所有演员、项目和公司名称均为产品演示内容。

后端接入建议采用：

- PostgreSQL
- 对象存储与视频转码服务
- 手机号或微信登录
- 实名认证服务
- WebSocket 或托管实时消息服务
- 消息已读状态与推送服务
- RBAC 项目团队权限

## 产品原则

- 演员基础建卡与接收邀请免费；
- 项目方为搜索效率和选角协作工具付费；
- 不出售演员联系方式；
- 不采用“付费保证进组”或虚假曝光；
- 公开资料、认证资料和私密资料分级管理；
- 系统通知与双方项目沟通分开；
- AI 仅辅助匹配和资料整理，不替代最终选角判断。
