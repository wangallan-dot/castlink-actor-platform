# 幕见 CastLink 产品说明

## 1. 产品定义

幕见是面向演员和剧组方的移动端剧组招募与演员发现平台。

演员自然地浏览招募，剧组自然地浏览演员；当用户产生明确目标后，再进入搜索、申请、邀请、沟通和试镜流程。

## 2. 核心身份

### 演员

- 浏览推荐和最新剧组招募
- 按地区、时限和项目类型主动搜索
- 维护演员卡、照片、视频和档期
- 申请招募或接受剧组邀请
- 围绕具体项目沟通并提交试镜

### 剧组方

- 浏览推荐、本地和档期开放演员
- 按地区、形象、技能和档期主动搜索
- 发布和管理剧组招募
- 接收演员资料或主动邀请演员
- 围绕具体招募沟通并管理试镜

## 3. 导航职责

### 首页顶部第一层

切换信息流来源：推荐、最新、当前地区。

### 首页顶部第二层

即时过滤当前信息流，不离开页面，也不要求二次确认。

### 底部导航

切换核心业务空间。中间发布入口根据演员或剧组身份显示不同操作。

### 招募和演员页面

用于有明确目标的主动搜索，提供完整搜索、地区、排序和筛选能力。

## 4. 流畅操作规则

- 打开详情后返回列表，恢复原滚动位置
- 地区、搜索词和筛选状态不丢失
- 点击当前底部标签时回到页面顶部
- 快捷分类点击后立即生效
- 普通消息未读只在消息入口显示，首页只展示重要业务待办
- 身份切换放在“我”页面，避免频繁干扰浏览
- 不允许脱离具体招募的大规模陌生私聊

## 5. 核心闭环

```text
剧组发布招募
→ 演员申请，或剧组邀请演员
→ 对方接受
→ 建立绑定招募和演员的会话
→ 剧组发送试镜任务
→ 演员提交 Self-tape
→ 剧组评审并更新结果
```

## 6. 平台原则

- 报酬、拍摄地区、日期和截止时间必须明确
- 演员联系方式分级开放
- 剧组身份和招募信息需要审核
- 当前阶段不设计任何商业化、会员或付费曝光功能
