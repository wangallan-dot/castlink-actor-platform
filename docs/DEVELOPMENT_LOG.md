# 开发记录

## 2026-07-17 — Mobile MVP 0.5.0

### 用户操作逻辑迭代

- 首页顶部调整为推荐、最新和当前地区，不再与底部业务入口重复
- 新增身份化第二层快捷分类，点击后立即更新信息流
- 首页改为连续发现流，招募和演员页面负责主动搜索
- 演员端底部改为：首页、招募、发布、消息、我
- 剧组端底部改为：首页、演员、发布、消息、我
- 中间发布按钮根据身份打开不同操作面板
- 身份切换移动到“我”页面
- 招募和演员页增加地区选择并保存筛选状态
- 点击当前底部标签可返回页面顶部
- 从详情返回列表时恢复滚动位置
- 移动端搜索入口按当前身份和页面自动确定搜索范围
- 版本升级至 `0.5.0`，PWA缓存升级至 `castlink-mobile-v5`

### 质量检查

- Oxlint：0 warnings，0 errors
- TypeScript：通过
- Vite production build：通过

## 2026-07-16 — Mobile MVP 0.4.0

### 产品结构重构

- 用户身份收敛为演员和剧组方
- “角色”栏目统一更名为“剧组招募”
- 增加推荐、剧组招募、演员顶部频道导航
- 增加全国和主要影视城市地区导航
- 演员端和剧组端使用不同的底部导航
- 登录后的首页改为待办事项和推荐信息流

### 剧组招募

- 增加申请截止时间、拍摄地区、具体区域、住宿和外地演员条件
- 增加正常、即将截止、今日截止、已截止状态
- 增加急招、今日截止、3天内和7天内筛选
- 剧组招募改为手机单列高信息密度卡片
- 已截止招募禁止继续提交演员卡

### 演员浏览

- 增加常驻城市、可工作地区、是否跨城和是否可驻组
- 增加推荐、本地、档期开放、最近活跃和已认证浏览模式
- 保留手机双列视觉卡片
- 删除专业版和商业化提示

### 质量检查

- Oxlint：0 warnings，0 errors
- TypeScript：通过
- Vite production build：通过

### 当前边界

- 身份切换、地区、收藏等使用浏览器本地状态
- 剧组招募和演员资料仍为演示数据
- 申请、邀请和消息尚未接入服务器
- 未接入真实登录、数据库、上传和推送
