# 幕见 CastLink

面向演员和剧组方的 **移动端优先** 剧组招募与演员发现平台。

在线预览：`https://wangallan-dot.github.io/castlink-actor-platform/`

当前版本：**Mobile MVP 0.5.0**

## 产品范围

当前只服务两个身份：

- 演员
- 剧组方

核心内容对象：演员资料、剧组招募。

核心流程：

```text
浏览 → 收藏 → 申请或邀请 → 沟通 → 试镜
```

当前阶段不考虑会员、套餐、广告、付费曝光和其他商业化功能。

## v0.5.0 用户操作结构

### 首页负责“逛”

顶部第一层切换信息流来源：

```text
推荐｜最新｜当前地区
```

顶部第二层对当前信息流进行即时过滤。

演员端：

```text
全部｜急招｜今日截止｜3天内｜短剧｜电影｜广告｜综艺
```

剧组端：

```text
全部｜本地｜档期开放｜最近活跃｜已认证｜动作｜方言
```

首页采用连续信息流，不要求用户先填写复杂筛选条件。

### 招募和演员页面负责“找”

- 招募页：搜索、地区、项目类型、截止时间和完整筛选
- 演员页：搜索、地区、档期、年龄感、形象和技能筛选
- 搜索词、地区和筛选条件保存在 URL 中
- 从详情返回列表时恢复原滚动位置

### 身份化底部导航

演员端：

```text
首页｜招募｜发布｜消息｜我
```

剧组端：

```text
首页｜演员｜发布｜消息｜我
```

中间“发布”会根据身份打开不同快捷操作。

身份切换移动到“我”页面，不再长期占用顶部导航。

## 主要功能

- 剧组招募单列信息流
- 演员双列视觉信息流
- 地区状态跨页面保持
- 招募截止时间与剩余时长
- 当前底部标签再次点击回到顶部
- 列表详情返回后恢复滚动位置
- 演员与剧组身份化快捷发布
- 项目消息中心
- PWA 安装和 GitHub Pages 自动部署

## 主要路由

| 路由 | 页面 |
|---|---|
| `/` | 身份化发现首页 |
| `/recruitments` | 剧组招募主动搜索 |
| `/recruitments/:roleId` | 剧组招募详情 |
| `/actors` | 演员主动搜索 |
| `/actors/:actorId` | 演员详情 |
| `/messages` | 项目沟通中心 |
| `/applications` | 演员申请与试镜 |
| `/recruitment/manage` | 剧组招募管理 |
| `/me` | 我的与身份切换 |
| `/profile/edit` | 演员资料编辑 |

## 当前技术边界

当前仍为可交互前端原型：

- 演员和剧组招募均为演示数据
- 身份、地区、收藏和资料草稿保存在浏览器
- 未接入真实登录和数据库
- 未接入真实图片、视频和附件上传
- 消息尚未接入服务器和跨设备同步
- 申请和邀请按钮尚未形成服务端业务状态
